# Activity Logs

Activity logs provide a complete audit trail of all changes made to records in the platform. Navigate to **System > Activity Logs** to view them.

## What Is Logged

Every create, update, and delete action on key resources is automatically recorded. Each log entry captures:

- **Causer** — The user who performed the action
- **Subject** — Which record was affected (e.g., a specific user, team, or role)
- **Event** — The type of action (created, updated, deleted)
- **Changes** — A comparison of the old and new values for updated fields
- **Timestamp** — When the action was performed

## Browsing Logs

The activity log table lets you browse through all recorded events. You can:

- **Search** for specific entries
- **Sort** by date or other columns
- **View details** of any entry to see the full list of changes

## Why Activity Logs Matter

Activity logs help you:

- **Track changes** — See who changed what and when
- **Audit access** — Review which users performed sensitive actions
- **Troubleshoot issues** — Trace back to the exact change that caused a problem
- **Meet compliance requirements** — Maintain a record of all data modifications

## Retention

Activity logs are retained indefinitely by default. Contact your system administrator if you need to configure a retention policy.
